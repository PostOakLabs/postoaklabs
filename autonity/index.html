// GLOSSARY

Terms used across all seven pages, defined once. "NTN" is short for Newton — noted here since the abbreviation is used throughout without being spelled out. This glossary grows as more pages are revised; if a term you need isn't here yet, it's likely still pending its page's own edit pass.

ACU
Autonity Currency Unit — the multi-currency basket the protocol targets. Not a stablecoin peg.
AFP
Autonomous Futures Protocol — Autonity's on-chain forecast-futures architecture. See Page 3.
AGC
Autonity Go Client — the reference node software (a fork of Geth) run by every peer on an Autonity network.
AOS
Autonity Oracle Server — the software validator nodes run to source external price data and submit price reports on-chain.
ASM
Auton Stabilization Mechanism — the CDP-based mechanism that stabilizes ATN's value against the ACU. ASMv3 refers to a planned, never-deployed free-floating version. See Page 2.
ATN
Auton — Autonity's native gas coin. An algorithmic stable asset targeting the ACU basket, not a fiat peg. See Page 2.
AutEx
The order-execution venue built on Autonity. See Page 5.
CDP
Collateralized Debt Position — a smart-contract lending mechanism where collateral (NTN or LNTN) is locked to borrow ATN. Used by the ASM.
DCC
Decentralized Clearing Contract — the on-chain contract that implements one derivatives product and holds every participant's position ledger for it.
Forecastathon
The four-season (S1–S4) forecast-futures competition series this archive documents. See Page 6.
LNTN
Liquid Newton — a validator-specific ERC-20 receipt token minted when NTN is delegated. Not interchangeable across validators; was planned as CDP collateral but never implemented.
NTN
Newton — Autonity's staking token. Bonded to validators to secure the network; also usable as CDP collateral.
OAFD
Oracle Accountability Fault Detection — outlier and misbehavior detection for validator oracle price submissions.
OFD
Omission Fault Detection — automated detection of validator inactivity via aggregated consensus signatures.

Full protocol reference: docs.autonity.org.
