Why this exists

An agent runtime needs a policy layer that audit can read.

Heuristic guardrails fail audit. Bespoke per-tenant policy doesn't scale. An off-the-shelf compliance vendor adds a service call to every agent action — and its output isn't schema-stable enough for the runtime to enforce deterministically.

The AINumbers + Post Oak Labs surface ships ~259 stateless, browser-side tools that already emit Policy Mandate v1 mandates, with an MCP tool definition on each. The runtime imports the manifest, the agent reads the mandate, the runtime enforces it. The audit log is the same JSON the agent received. No vendor call. No serialization gap. No bespoke policy DSL.

The six scenarios in this hub walk an agent team through that adoption: design the policy surface, sandbox the mandate, then apply it across the two most-litigated fintech use cases (AML and BaaS). The cross-links pin the policy boundary to its actual regulatory and asset context.

For the acquirer reader A deterministic, schema-stable policy substrate is a strict prerequisite for agentic-payments scale. The catalog is the IP — the scenarios are the proof that it composes.

Six tools underneath the scenarios.

Each scenario is a packaged surface; the underlying AINumbers tools are where the runtime team will go to dig in. AP2 schemas and MCP manifests are exposed per-tool.

  • T102 ↗
    AP2 Payments Checker
    Validates an Policy Mandate v1 mandate against the canonical schema; surfaces missing intent and constraint fields before the runtime enforces.
  • T131 ↗
    AP2 AML Mandate Builder
    Builds AP2 mandates with the AML-specific constraint set — beneficial-owner gates, sanctions guards, TM referrals. The tool underneath Scenario #23.
  • T164 ↗
    AP2 BaaS Mandate Builder
    BaaS-specific mandate composition — sponsor-bank approval gates, programme-policy slots, BIN-tier ceilings. The tool underneath Scenario #24.
  • RBE-04 ↗
    Agent Guardrail Builder
    Guardrail composer for any agentic surface — declarative limits compiled to AP2 mandate JSON the runtime can enforce without code change.
  • RBE-06 ↗
    Agentic Mandate Sandbox
    Sandbox a mandate against simulated MCP discovery and agent execution — pre-deploy sanity check before the mandate touches production traffic.
  • RBE-07 ↗
    Signal Auditor
    Replay-and-trace tool for agent decisions — inspects the signal chain that led to a runtime guard firing. Useful for incident review and audit prep.
