Why this exists

The BaaS economics question is the same question every time — just with different inputs.

Sponsor-bank passthrough, programme agreement terms, BIN sponsorship tiers, float revenue under a given rate path, BaaS comparator economics — these all collapse to the same calc. Every BaaS evaluation rebuilds the spreadsheet from scratch. Every embedded-lending product team relearns origination economics. Every programme manager renegotiates VRP mandate parameters that should be the AP2 standard.

The catalog's BaaS cluster is the curated answer to that — modeled in the browser, exported as an AP2 programme-policy mandate the runtime can enforce. Six demos walk the buyer from "which provider" to "what mandate does my agent runtime read at execution time" — the last cell in the spreadsheet that always gets left out.

The cross-link demo (24) is the same surface the Agentic Runtime hub exposes — because the BaaS team's policy and the agent-runtime team's policy converge at this mandate format.

For the acquirer reader A BaaS surface that exports policy as AP2 mandates is the way a buyer plugs embedded-finance customer demand into an agentic future without rebuilding the programme-policy layer.

Seven tools underneath the BaaS demos.

The catalog's BaaS cluster (T152–T164) plus the rules-based engines (RBE-08, RBE-13). AP2-exportable, MCP-discoverable, none stateful.

  • T152 ↗
    BaaS Provider Comparator
    Comparator across BaaS providers — fee structure, ledger model, sponsor-bank network, regulatory passthrough. The tool underneath Demo #14.
  • T154 ↗
    Ledger Architecture Builder
    Builds the ledger architecture for a BaaS programme — sub-ledger granularity, FBO account structure, reconciliation cadence, audit-trail surface.
  • T155 ↗
    BIN Sponsorship Economics
    Models BIN sponsorship tier economics — issuer fees, scheme passthrough, tier-rate ceilings. The fast-second model a BaaS programme runs after the comparator.
  • T156 ↗
    Wallet Float Yield Estimator
    Float-yield projection across interest-rate scenarios — programme deposit aggregation, MMF allocation, sweep window optimisation. The tool underneath Demo #17.
  • T160 ↗
    Embedded Lending Unit Economics
    UE model for embedded-lending products — origination cost, default curves, capital cost. The tool underneath Demo #12.
  • RBE-08 ↗
    Fee Schedule Engine
    Declarative fee-schedule composition — passthrough vs. markup, tiered breakpoints, volume waivers. Exports as AP2 mandate the BaaS runtime can apply.
  • RBE-13 ↗
    Interchange Qualification Engine
    Qualification logic for interchange tiering — MCC, data quality, settlement window. The tool every BaaS programme needs to predict interchange capture.
